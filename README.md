## 🎯 这是做什么的

> 在观看长篇网页视频（如技术教程、企业培训、公开讲座）时，用户经常会被各种糟糕的播放体验打断：
>
> 1. **频繁的“防挂机”强制互动：** 视频播到一半，突然弹出“AI提问”、“您还在看吗”等必须手动点击才能继续的弹窗。
> 2. **常规浏览器插件易失效：** 各大视频网站广泛采用前端混淆、Shadow DOM 和强力的底层行为检测（如 `isTrusted` 验证），导致普通的“自动连播”插件极易失效甚至被封禁账号。
> 3. **播放状态易假死：** 遇到网络卡顿或无形遮罩时，传统脚本容易陷入死循环，无法自动重启播放流。
>
> **VideoFlow (智流)** 旨在打造一个通用的“视频播放状态守护者”。它采用策略模式与底层物理模拟技术，能够在不触碰网站安全底线的前提下，优雅地处理各类视频弹窗。

## 🔥 痛点与定位

### 市面上的方案为什么不靠谱？

| 方案 | 缺陷 | 后果 |
|---|---|---|
| 写死 DOM 的 Python 脚本 | 绑定前端实现细节 | 平台重构 → 脚本报废 |
| JS 注入抢占式工具箱 | 高速发包、伪造事件 | `isTrusted` 检测 → 封号 |
| 鼠标连点器 + 定时器 | 机械操作无随机性 | 几个循环就触发风控 |
| OCR + 截图对比 | 环境依赖重，不稳定 | 文字识别失败就卡死 |

### VideoFlow 的解法

- **不写死选择器：** 策略模式解耦，适配器即插即用。平台改版只需更新适配器，核心引擎不动。
- **不走 JS 注入：** 所有操作走 Playwright 系统级模拟（鼠标轨迹、真实点击、自然间隔），行为特征与真人无异。
- **不死循环：** 内置异常捕获 + 慢车道降级，遇到未知弹窗就唤醒 VLM Agent 分析解决，而不是崩溃退出。

## ✨ 核心特性

手动登录后、无需配置选择器、无需写任何代码。自动拉出浏览器，自动播放，自动下一个视频。

支持**任务队列模式** —— 一次性导入多个链接，排队逐个完成。

### 🛡️ 行为引擎：操作像真人，过检不标记

| 维度 | 实现方式 | 效果 |
|---|---|---|
| 鼠标移动 | 贝塞尔曲线轨迹 + 随机偏移 | 不是瞬移，不是直线，是「人」的手 |
| 点击操作 | Playwright 原生 `.click()` + 物理坐标 | 携带 `isTrusted: true`，无法被 JS 检测伪造 |
| 操作间隔 | 随机化 Human Delay（0.8s ~ 2.5s） | 不是固定间隔，不触发频率检测 |
| 浏览器指纹 | 真实 Chromium 实例，非 CDP 远程 | 无头也带完整指纹 |

**一句话：** 该快的快（播放流程），该慢的慢（操作节奏），该随机的随机（鼠标轨迹）。

### 🧠 自愈架构：改版不崩，自己想办法

| 路径 | 触发条件 | 行为 |
|---|---|---|
| **快车道（Fast Path）** | 常规流程正常 | 硬编码的特征匹配（物理坐标、稳定选择器），毫秒级轮询，零成本 |
| **慢车道（Slow Path）** | Fast Path 报错 | **不崩溃！** 降级唤醒 VLM Agent → 分析屏幕截图 + DOM 树 → 自动定位新元素 → 继续运行 |

> 平台改版那天，你的工具不会瘫在那里等你修。它会说：「让我看看你改成什么样了。」

## ❓ FAQ

**Q：这东西用来干什么？**  
A：任何有视频播放页面的网站。输入链接，自动播放，其中的广告/弹窗都可自动关闭。视频具体是什么内容，取决于你给了什么链接。

**Q：VLM Agent 用外部 API 会不会很贵？**  
A：仅 Fast Path 失败时才调用，正常运行零消耗。截屏分析间隔 5 秒一次，日常使用几乎不触发。
